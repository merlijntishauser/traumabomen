# i18next 26 + react-i18next 17 Upgrade Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade i18next from 25 to 26 and react-i18next from 16 to 17, closing Dependabot PR #69.

**Architecture:** Both packages must be upgraded together because react-i18next 17 requires i18next >= 26.0.1 as a peer dependency. The project's i18next usage is minimal and clean: no custom formatters, no removed APIs, and all `<Trans>` components use explicit `i18nKey` props. This makes the upgrade a version bump with verification, not a code migration.

**Tech Stack:** i18next 26, react-i18next 17, i18next-browser-languagedetector 8, i18next-http-backend 3

---

## Breaking Changes Analysis

### i18next 26

| Breaking change | Used in project? | Action |
|-----------------|------------------|--------|
| `interpolation.format` function removed | No (not in `src/i18n.ts` or anywhere) | None |
| `initImmediate` option removed | No | None |
| `showSupportNotice` option removed | No | None |
| `i18next.format()` routes through Formatter API only | No direct `i18n.format()` calls | None |

### react-i18next 17

| Breaking change | Used in project? | Action |
|-----------------|------------------|--------|
| `transKeepBasicHtmlNodesFor` serialization fix for auto-generated keys | No: all 3 `<Trans>` usages have explicit `i18nKey` | None |
| Peer dependency: `i18next >= 26.0.1` | Must upgrade together | Bump i18next first |

### Plugin compatibility

| Plugin | Current version | i18next 26 peer dep | Action |
|--------|----------------|---------------------|--------|
| `i18next-browser-languagedetector` | 8.2.1 | No peer dep on i18next | None |
| `i18next-http-backend` | 3.0.2 | No peer dep on i18next | None |

### Trans component inventory (all safe)

All 3 uses have explicit `i18nKey`, so the auto-generated key serialization change does not apply:

1. `src/components/AppFooter.tsx:48` -- `i18nKey="mentalHealth.footer"`, object components syntax
2. `src/components/MentalHealthBanner.tsx:28` -- `i18nKey="mentalHealth.banner"`, object components syntax
3. `src/components/PasswordStrengthMeter.tsx:28` -- `i18nKey="password.managerHint"`, array components syntax
