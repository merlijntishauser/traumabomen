# Spike: crypto compatibility for the iOS companion (M1)

Date: 2026-07-05
Design: `docs/plans/2026-07-05-ios-companion-design.md`
Verdict: **GO. The KMP-core premise holds.**

## Question

Can the native stack planned for the KMP core (libsodium `crypto_pwhash`
for Argon2id, Apple CryptoKit for AES-256-GCM) reproduce the web client's
zero-knowledge crypto byte-for-byte, in both directions?

## Method

1. The fixture (canonical home: `frontend/src/fixtures/crypto-compat.fixture.json`,
   also asserted by `frontend/src/lib/cryptoCompat.unit.test.ts` and the KMP
   core suite) was generated by the *real* web stack: the committed
   `argon2-bundled.min.js` (argon2-browser 1.18.0) plus WebCrypto, executed
   in Chromium against the dev server, mirroring `frontend/src/lib/crypto.ts`
   call-for-call. It contains the passphrase, salt, expected master key,
   an encrypted key ring, and encrypted person and journal blobs.
2. `verify_libsodium.py` (pynacl = libsodium bindings, the same C library
   the KMP core will bind on both platforms) re-derives the master key and
   walks the full decrypt chain, then encrypts a fresh blob.
3. `verify_cryptokit.swift` (CryptoKit, the iOS `actual`) walks the same
   decrypt chain and encrypts a fresh blob.
4. Both fresh blobs were decrypted back in the browser by WebCrypto using
   the web's exact `decryptFromApi` steps.

## Results

| Check | Result |
|---|---|
| libsodium Argon2id (opslimit=3, memlimit=64 MiB, ALG_ARGON2ID13) == argon2-browser derivation | PASS, byte-identical |
| libsodium side: key ring -> tree key -> person/journal decrypt | PASS |
| CryptoKit: same chain from raw master key | PASS |
| libsodium-encrypted blob decrypted by WebCrypto | PASS |
| CryptoKit-encrypted blob decrypted by WebCrypto | PASS |
| SHA-256 passphrase hash (lock-screen verify) | PASS |

Timing: Argon2id 64 MiB / t=3 / p=1 in native code runs ~490 ms on an
M-series Mac. Expect roughly 1 to 2 s on recent iPhones: acceptable for the
weekly passphrase re-verify, and irrelevant day-to-day since Face ID unwraps
a cached key without any derivation.

## Findings worth carrying into M2

- libsodium's parameter mapping is direct: `opslimit` = web `time`,
  `memlimit` = web `mem` * 1024. No translation layer needed.
- WebCrypto appends the 16-byte GCM tag to the ciphertext; CryptoKit splits
  it back out via `SealedBox(nonce:ciphertext:tag:)`. The KMP common code
  should define the blob as `iv || ciphertext+tag` and let each `actual`
  handle its library's tag convention.
- Swift's `JSONEncoder` escapes `/` as `\/`. Valid JSON, and the web parses
  it fine (proven), but the KMP core will use kotlinx.serialization; add a
  fixture assertion for escaping behavior anyway.
- The fixture in this directory is the seed of the M2 shared compatibility
  suite: Vitest and the KMP tests must both assert it.

```
python3 -m venv /tmp/spike-venv && /tmp/spike-venv/bin/pip install pynacl cryptography
/tmp/spike-venv/bin/python verify_libsodium.py

```

last run, kept so the browser-side decrypt step is reproducible as-is.
